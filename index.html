Portfólio

Banpará

Resumo:

Durante o meu estágio, eu e outro estagiário recebemos a tarefa de desenvolver um sistema web que automatizasse a validação de contratos com base em pontos de função. Os contratos eram entregues em uma planilha do Excel com um formato pouco agradável e difícil de ser analisado pelos profissionais responsáveis. Com o novo sistema, o processo de validação se tornou mais rápido e eficiente, facilitando o trabalho dos analistas e garantindo maior precisão na avaliação dos contratos. Além disso, a automatização do processo permitiu uma redução significativa de erros humanos e aumentou a produtividade da equipe.

Funcionalidades:

  • - Telas de cadastro e login com JWT. para garantir a segurança e a privacidade das informações.
  • - Controle e gestão de usuários. Isso permite adicionar, editar, visualizar e excluir usuários com facilidade.
  • - Leitura de planilhas, com a capacidade de fazer upload e download de arquivos via buffer. Isso possibilita a importação e exportação de dados de forma simples e eficiente.
  • - Controle e gestão de fornecedores e sistemas, para garantir que todas as informações sejam gerenciadas de forma adequada e segura. Isso inclui a possibilidade de adicionar, visualizar e excluir fornecedores e sistemas terceirizados do banco.

Tecnologias usadas:

  • ASP.NET Core
  • Dapper
  • SQL Server
  • Blazor WASM
  • JWT Token

Instalador SA

Resumo:

Instalador SA é uma API desenvolvida para um aplicativo de gestão de orçamentos para instaladores e revendedores credenciados na Eletro Transol. À medida que o aplicativo cresceu, tornou-se necessário atualizar o sistema, migrando-o do Firebase para o ASP.NET Core, para melhorar o desempenho e a segurança, que são recursos oferecidos pelo .NET. Como responsável pela migração, fiquei encarregado de trabalhar no back-end da API, que não possui uma interface visual direta. Link do App: Front-End Desenvolvido por Diemisom Melo (Exodus)

Funcionalidades:

Desenvolvi todas as funcionalidades do aplicativo para usuários, clientes, instaladores, vendedores e administradores. Dado que o projeto apresentava uma complexidade significativa, decidi dividi-lo em microsserviços, de modo que cada um pudesse funcionar independentemente dos outros. Isso melhorou significativamente o desempenho do sistema e a velocidade das respostas.

O sistema incluía recursos de cadastro e login de usuários com JWT Token, além de recursos de autorização que controlavam as ações permitidas para cada usuário. Segui boas práticas de desenvolvimento, como Clean Code e Arquitetura SOLID, para garantir a qualidade do código e torná-lo mais fácil de manter e modificar no futuro.

Tecnologias usadas:

  • ASP.NET Core
  • EF Core
  • SQL Server
  • AutoMapper
  • JWT Token
  • Azure Services
